# Persistent repo memory

- This is a PUCV Optimización Computacional project.
- Goal: rigorous partial replication of Duran-Mateluna, Ales & Elloumi (2023), “An efficient Benders decomposition for the p-median problem”.
- Verified local core: Benders/F4-style lazy-cut method derived from F3.
- Implemented core: master variables `y_j` and `theta_i`; objective `min sum_i theta_i`; constraint `sum_j y_j = p`; Phase 1 LP relaxation; Phase 2 binary branch-and-Benders-cut; Gurobi lazy callbacks; separator using sorted sites `S_i` and `ktilde_i`.
- Verified local experiments before this branch: `toy1`, OR-Library `pmed1`–`pmed15`, `rl1304` Table 2 subset, `kroA100`, warm/cold comparison, separator tests.
- Not implemented / not run: Zebra, PopStar, constraint reduction, reduced-cost fixing, monolithic C benchmark, BIRCH, RW large, ODM, huge TSP.
- Mandatory evidence rule: every numerical claim must be backed by `results/*.csv`, `results/logs/*.log`, or tests.
- Forbidden claims unless new evidence is added:
  - “we outperform Zebra”
  - “we replicated the full paper”
  - “we implemented PopStar”
  - “we compared against a monolithic C baseline”
  - “we ran huge TSP/BIRCH/ODM”
- Time-limit rule: if a result is described as 300s / 5 minutes, it must be run through a wrapper enforcing `timeout=300`.
